MySQL

2.1 数据库简介

​ 作用: 高效的存储和管理数据

​ 特点: 持久化、高效、保证数据的有效性

​ 分类: 关系型<二维>和非关系型

关系型概念:

​ 字段< 记录< 表 table < 数据库 database

数据库最终还是将数据保存到磁盘文件中

关系型数据库系统RDBMS： 数据库客户端、数据库服务器

SQL 作用: 客户端和服务器用来通信的标准语言

select * from students;

SELECT * FROM students;

SQL 中关键词不区分大小写; 给别人看建议关键词大写 

# 1 安装 mysql 数据库  了解
linux> sudo apt-get install mysql-server mysql-client

# 2 通过系统命令查看 mysql 是否已经启动
linux> ps aux | grep mysql

# 3                状态     启动  停止  重启
linux> sudo service mysql status /start/stop/restart

# 4 连接 mysql
linux> mysql -uroot -pmysql
linux> mysql -uroot -p回车再写密码 密码看不见

# 5 退出登录 
mysql> exit/quit/ctrl d

2.2 数据类型⭐️⭐️⭐️⭐️

年龄 整数类型 99 AAA

用来保证数据有效性的强制约束、

数据类型决定了的数据解析方法、数据存储空间大小

“99” 99

bit、decimal、char、varchar、smallint、enum

选择类型-重点

​ 大范围小范围 看数据的范围 尽量选小

​ 定长char 变长varchar 看数据的长度是否固定

​ 有符号无符号(unsigned) 看数据是否存在负数

2.3 约束⭐️⭐️⭐️⭐️

  • 主键约束 primary key

    根据主键唯一确定一行记录 对上号

  • 非空约束 not null 不允许为空值 null

  • 唯一约束 unique 不允许字段重复

  • 默认约束 default 如果用户没有提供值 使用默认值;

  • 外键约束 foreign key
    一个表使用来自另一个表的数据
    前者称子表 后者称主表
    将子表中 kongfuid 定义为外键
    外键约束: 当修改外键字段值时需要保证 该值必须在主表字段中存在；否则报错

2.4 数据库操作⭐️⭐️⭐️

-- 展示当前系统中所有的数据库
mysql> show databases;

-- 创建数据库 指定字符集  默认拉丁1不能存中文
❌mysql> create database py29 charset=utf-8;
mysql> create database py29 charset=utf8;
-- 切换到指定数据库使用
mysql> use py29;
-- 查看当前正在使用的数据库   如果use过返回NULL
mysql> select database();
-- 展示创建数据库的 SQL 语句
mysql> show create database py29;
-- CREATE DATABASE `py29` /*!40100 DEFAULT CHARACTER SET utf8 */

-- 慎重 删库
mysql> drop database 数据库名;

2.5 表结构操作⭐️⭐️⭐️

表中需要几个字段 什么名字 每个字段需要什么类型 约束 

-- 展示当前数据库中所有的表
mysql> show tables;

-- 创建表
-- create table 表名(字段名 类型 [约束],...);
mysql> create table students(
        id int unsigned primary key auto_increment,
        name varchar(32) not null,
        age tinyint unsigned default 18,
        height decimal(5,2),
        gender enum('男','女','保密')
        );
-- 查看表结构    describe 描述   
mysql> desc students;
-- 展示创建表的 SQL 语句
mysql> show create table students;

-- 删除表 - 慎重
drop table 表名;
-- 修改表结构 alter alert⭐️⭐️
-- alter table 表名 
-- 1 添加字段alter table 表名  add 字段名 类型 约束;
alter table students add birthday datetime not null;
-- 2 修改字段类型和约束
alter table students modify birthday date not null;
-- 3 修改字段名 字段类型和约束
alter table 表名 change 原字段名 新字段名 类型 约束;
alter table students change birthday birth date ;
-- 4 删除字段
alter table students drop 字段名;
alter table students drop birthday;

如果多行 SQL 语句写错了

ctrl c 回车终止该语句 上键调出来继续写

3.6 表数据操作⭐️⭐️⭐️⭐️⭐️

-- 1 查询数据 select 字段名,.... from 表名;
-- * 通配所有字段
mysql> select * from students;
select id,name,age,height,gender from students;
select name,age,height from students;

-- 2 插入数据 
--insert into 表名(字段名,....) values (值1,...),(值1,...)
-- 2.1 全列插入  数据和表中字段  顺序数量完全一致的
insert into students(id,name,age,height,gender) values (1,"张三",18,175.88,'男');
insert into students(id,name,age,height,gender) values (0,"张三丰",150,178.00,'男'),(0,'张无忌',25,170,'男');
-- 全列插入可以省略表名后面的字段列表  id=0表示自动递增  null
insert into students values (0,'赵敏',20,165,'女');

-- 2.2 部分列插入 
insert into students(name,height,gender) values ('周芷若',165,'女');
insert into students(name,height,gender) values("陈友谅",175,'保密');
-- 3 修改数据   update 表名 set 字段名=值 where条件;
-- where条件是筛选出满足条件的记录
mysql> update students set age=28 where id = 1;
mysql> update students set height=160 where id = 5;
mysql> update students set age=28,height=166 where id = 4;

-- 4 删除数据  delete from 表名 where条件;
mysql> delete from students where id = 6;

delete 方式称为物理删除 很难恢复

逻辑删除: 在这个表新增一个用以标识记录是否被逻辑删除了

逻辑删除实现步骤

  • 新增一个字段用以标识记录是否被逻辑删除了
  • 逻辑删除 将字段值修改 1
  • 撤销逻辑删除 将字段值修改 0

逻辑删除本质上是标记操作

alter table students add is_delete bit not null default 0;
update students set is_delete = 1 where 条件
update students set is_delete = 0 where 条件

3.7 as关键字⭐️⭐️⭐️⭐️

as 作用: 起别名 只在当前 SQL中使用

as 使用形式: 原有名称 as 别名

-- 将 id 起别名  name 起别名
mysql> select id as '学号',name as '姓名' from students;

mysql> select id as '学号',name as '姓名',height as '身高' from students;

-- 还可以给表名起别名
❌select students.id,students.name from students as s;
mysql> select s.id,s.name from students as s;

作用: 去除结果集中的重复行

形式: select distinct 字段名,…

|  120 | 男     |

|  120 | 女     |
select distinct age,gender from students

3.9 where条件⭐️⭐️⭐️⭐️⭐️

where 条件作用 筛选出满足条件的记录进行下一步操作

-- > < = >= <= 
mysql> select * from students where height > 167;
mysql> select * from students where height >= 166;

-- != <> 不等
mysql> select * from students where gender != '女';

-- 逻辑运算符  and or not 
mysql> select * from students where age>50 and height>170;
mysql> select * from students where height>175 or gender='女';

-- 模糊查询 只知道信息的一部分 查询
-- % 类似* 任意个任意字符  _类似? 一个任意字符
-- 姓张的
select * from students where name like '张%'
-- 张三后面任意个任意字符
select * from students where name like '张三%'
-- 张三后面一个任意字符
select * from students where name like '张三_'
-- 张后面2个任意字符
select * from students where name like '张__'

-- 范围查询
-- 连续范围查询  between 下限 and 上限
select * from students where age >=50 and age <= 120;
select * from students where age between 50 and 120;
-- 不连续范围查询  字段 in ()
mysql> select * from students where age=28 or age=38 or age=48 or age=58;
mysql> select * from students where age in (28,38,48,58);

-- 判空操作 IS NULL    IS NOT NULL 非空
-- NULL 是是一种空状态 不能比较
insert into students(name)values('天山童姥');

❎select * from students where height=NULL;
mysql> select * from students where height IS NULL;
❎select * from students where height NOT IS NULL;
mysql> select * from students where height IS NOT NULL;
