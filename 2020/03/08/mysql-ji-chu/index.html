1.3 约束⭐️⭐️⭐️⭐️

  • 主键约束 primary key

    根据主键唯一确定一行记录 对上号

  • 非空约束 not null 不允许为空值 null

  • 唯一约束 unique 不允许字段重复

  • 默认约束 default 如果用户没有提供值 使用默认值;

  • 外键约束 foreign key
    一个表使用来自另一个表的数据
    前者称子表 后者称主表
    将子表中 kongfuid 定义为外键
    外键约束: 当修改外键字段值时需要保证 该值必须在主表字段中存在；否则报错

2.1 排序

作用: 展示数据集的时候根据一项或者多项数据按照一定顺序进行排列

多个字段排序: 前几个字段相同就使用后续字段继续排序

语法格式: order by 字段名 排序方式,字段名 排序方式,…..

默认排序方式就是升序ASC、降序 DESC descend 向下倾斜

-- 默认输出
select * from students;

-- 用身高排序
select * from students order by height;
select * from students order by height ASC;
select * from students order by height DESC;

-- 先按年龄升序排  年龄相同的就按身高升序排
select * from students order by age ASC,height ASC;

-- 先按年龄升序排 年龄相同的就按身高升序排  身高相同的再按 id 降序排
select * from students order by age ASC,height ASC,id DESC;

2.3 聚合函数

聚合函数又叫组函数: 将一组数据运算(聚合)出一个值

如果字段值为 NULL 则不参与聚合运算

用法: 聚合函数(字段名)

常用聚合函数: 和sum、数量count、平均数 avg、最大值max、最小值min 

-- 求出班级中所有同学的平均年龄
select avg(age) from students;

-- 求出班级中所有同学的平均身高
select sum(height)/count(height) from students;
select avg(height) from students;

-- 求出班级中最大年龄
select max(age) from students;
-- 求出班级中最小年龄
select min(age) from students;

-- 求出班级中学生的数量
select count(*) from students;

-- 求出班级中学生的身高综合
select sum(height) from students;

-- 求出班级中最大年龄和最高身高
select max(age),max(height) from students;

2.4 分组

作用: 将数据按照多个字段进行分组、对产生的每个小的分组进行分别的统计运算。

字段相同的就在一个组。不同的就在不同组。

形式: group by 字段名;

-- 分组之后 数据形式发生变化 不能支持使用 SQL 查看  多维结构 
❎select * from students group by gender;

-- 1 group_concat(字段) 将分组内每个成员的字段值拼接在一起显示
-- 显示每种性别 及其每个性别所有的人名 每个人的id
mysql> select gender,group_concat(name),group_concat(id) from students group by gender;

-- 2 分组结合聚合函数
-- 默认情况下聚合函数是把整个表当做一个组进行计算
-- 一旦分组之后 聚合函数针对每个分组分别进行计算
select gender,avg(age) from students group by gender;
-- 按性别分组 求出每组平均年龄 每组中所有年龄拼接显示
mysql> select gender,avg(age),group_concat(age) from students group by gender;
mysql> select gender,group_concat(name) from students group by gender;

-- 能够放到 select 后面的字段 要么就是 group 后的字段 要么放在聚合函数中  要么放在 group_concat()

-- 3 分组条件 只能使用 having 过滤分组结果中的数据
mysql> select gender,avg(age) from students group by gender;
mysql> select gender,avg(age) from students group by gender having avg(age) > 50;

select gender,avg(age) from students group by gender having count(*) < 2;

-- 4 with rollup 汇总行 
-- 将整体数据再交给 select 后面的函数再运算一遍求出结果 
mysql> select gender,avg(age) from students group by gender with rollup;
| gender | avg(age) |
+--------+----------+
| NULL   |  18.0000 |
| 男     |  69.5000 |
| 女     |  74.0000 |
| NULL   |  63.4286 | 汇总行
+--------+----------+

-- with roll 和 having 顺序 
mysql> select gender,avg(age) from students group by gender with rollup having avg(age) > 50;

2.7 子查询

需求: 求出班级中年龄大于平均年龄23岁的学生信息

​ 1 求出平均年龄 select avg(age) from students;

​ 2 求解 select * from students where age>平均年龄;

概念: 子查询:嵌套在内部的 select 语句 使用子查询的查询语句:主查询

形式: 主select 语句 条件(子select 语句)

-- 分类
--1 标量子查询  子查询返回的结果是一个值(一个字段)
-- 求出班级中身高高于平均身高的学生信息
-- 第一步 求出平均身高
select avg(height) from students;
-- 第二步  使用第一个的数据进行外部查询
mysql> select * from students where height > 170.98;
-- 子查询写法
mysql> select * from students where height > (select avg(height) from students);

--2 列子查询  子查询返回的结果是     一列多行
-- 查询 正在被英雄使用的所有技能名称
-- 第一步 求出所有英雄正在使用的技能 id
mysql> select gongfuid from hero;
-- 第二步 使用第一步求出的技能 id 去技能表中进行比较
select name from gongfu where id in (1,2,3,5);
-- 子查询写法
mysql> select name from gongfu where id in ( select gongfuid from hero);

--3 行子查询   子查询返回的结果是  一行多列
-- 查询在班级中  身高最高并且年龄最大的学生信息
-- 第一步 求出最高身高和最大年龄 
select max(height),max(age) from students;
-- 第二步 使用第一步求出的数据 去学生表中比较
mysql> select * from students 
    where height=178 and age = 150;
mysql> select * from students 
    where (height,age) =(178,150);
-- 子查询写法 
mysql> select * from students where (height,age) =(select max(height),max(age) from students);

3.1 外键约束⭐️⭐️⭐️⭐️

作用: 约束时子表在更新、插入外键字段的时，该值一定要在主表中存在;否则更新、插入失败

-- 1 给已经存在的子表中一个字段加上外键约束
alter table 子表名 add foreign key(字段名) references 主表名(字段名);
alter table hero add foreign key(gongfuid) references gongfu(id);

-- 2 查看已经创建好的外键约束<外键约束的名字>
mysql> show create table hero;

  `id` int(10) unsigned NOT NULL AUTO_INCREMENT,
  `name` varchar(64) NOT NULL,
  `gongfuid` int(10) unsigned NOT NULL,
  PRIMARY KEY (`id`),
  KEY `gongfuid` (`gongfuid`),
  CONSTRAINT `hero_ibfk_1` FOREIGN KEY (`gongfuid`) REFERENCES `gongfu` (`id`)
)

理解设计范式、ER 模型中关系 理解思想

-- 数据库设计  存储系统中需要几个表 每个表分别有什么字段 表之间有什么关系

-- NF normal form范式: 规律、规则
-- 1NF 一个字段不允许包含多种数据

-- 2NF: 1NF;表中有主键;非主键完全依赖于主键(不能出现部分依赖
非主键字段  商品单价、商品名称 只依赖于主键的部分   商品id，不依赖于订单 ID  形成了部分依赖   导致不满足完全依赖)

-- 3NF:2NF 非主键直接依赖于主键（而不能传递依赖于主键
非主键字段 用户名字 地址 城市 只依赖于非主键字段用户 ID
用户 ID 再依赖于主键订单 ID
形成了  非主键字段 用户名字 地址 城市 对主键的传递依赖
